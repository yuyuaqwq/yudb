## yudb

基于B+树的高性能持久化键值数据库

## 特性
- 完全可序列化事务(ACID)
- 读写并发(MVCC)
- 存储可变长度KV
- 单数据库文件最大支持4TB
- B+树节点内嵌红黑树以提高写性能
- 结合 Shadow Paging(影子分页) 和 Write-Ahead Log(预写式日志) 实现高性能读写

## 设计

- 读性能：
  - 基于Cow B+树，实现无锁并发读，通常情况下性能基本接近内存B+树。

- 写性能：
  - 为避免写性能下降到一个较难接受的程度，引入了wal。
  - bolt、lmdb
  - 由于其事务提交时要求脏页落盘，即刻存在较多随机写。
  - leveldb
  - 之所以写入迅速的原因是，在写入时只会修改内存跳表，并追加wal，不存在随机写。
  - yudb
  - 页缓存修改 + wal 策略，且由于是Cow B+树，只需记录redo，而无需记录undo。

- 补充
  - 持久化介质同步慢是无法在应用层面上解决的；
  - 故默认策略是不会在每次追加wal后就保证wal落盘，而是存放到文件系统的缓存当中，等待一定时间后批量落盘。
  - 弊端就是，遇到 断电/操作系统 崩溃等情况，可能会丢失一部分未来得及落盘的数据。
