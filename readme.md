## yudb
基于B+树结构的高性能持久化kv数据库

## 特性
- 支持事务(完全ACID语义)
- MVCC
- 读并发，写串行，读写互不阻塞
- 存储可变长度kv(开发中)
- 结合 Shadow Paging(影子分页) 和 Write-Ahead Log(预写式日志) 实现高性能读写

## 性能简述

- 读性能简述：
  - 基于Cow B+树，实现无锁并发读，性能基本接近内存B+树。

- 写性能简述：
  - 若单纯基于Cow B+树实现，写性能会下降到一个较难接受的程度
    - 如bolt，尽管lmdb未做过测试，但在理论上由于其事务提交时要求脏页落盘(随机写)，也绝对不会快
  - leveldb写入快的原因是写入时只会修改内存结构，并追加wal，不存在随机写。
  - yudb是 页缓存修改 + wal 策略，且由于是Cow B+树，只需记录redo，而无需记录undo。

- 补充
  - 持久化介质同步慢是无法在应用层面上解决的；
  - leveldb之所以能达到如此高的写性能最根本的原因，也是因为其并不会每次追加wal后就保证wal落盘，而是存放到文件系统的缓存当中，等待一定时间后批量落盘。
  - 弊端就是，遇到 断电/操作系统 崩溃等情况，可能会丢失一部分未来得及落盘的数据。
